# 案例：在不支持 OAuth 的平台上读取飞书电子表格

## 背景

某同事在内部平台中配置了飞书技能集（feishu skills），希望通过 Agent 读取飞书电子表格内容。但该平台不支持 OAuth 认证流程，导致无法使用需要用户授权的 API。

## 问题

- 飞书技能集默认通过 OAuth（`user_access_token`）认证，调用 API 时会弹出浏览器要求用户登录授权
- 内部配置平台无法完成 OAuth 交互流程
- 飞书官方 MCP 虽然可用，但只能读取表格元信息，无法读取实际内容，且不支持功能扩展

## 解决方案

### 1. 使用应用权限代替用户权限

飞书技能集中的部分功能已支持**应用级权限**（`tenant_access_token`），无需 OAuth。例如发送消息功能的调用方式：

```python
api_request("POST", "/im/v1/messages", ..., use_user_token=False)
```

参考这一模式，让 Agent 自行改造电子表格读取的 skill，将 `use_user_token` 设为 `False`，改用应用级 token 调用 API。

### 2. 通过建群解决文档权限问题

使用应用权限后，遇到了文档访问权限不足的问题——应用默认无法访问用户个人目录下的文档。

解决方法：**创建一个飞书群，将文档分享到群中**。应用作为群成员即可获得文档的访问权限，无需额外配置复杂的权限审批。

## 总结

| 问题 | 解决方式 |
|------|---------|
| 平台不支持 OAuth | 改用 `tenant_access_token`（应用权限） |
| 应用无法访问个人文档 | 建群分享文档，应用通过群成员身份获取权限 |
| 飞书 MCP 功能不全 | 使用 feishu skills 自行扩展所需功能 |
